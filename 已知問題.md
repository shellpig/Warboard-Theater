## 問題 3:兵力數字常卡在整數,缺乏持續變動

### 現象

- 部隊名牌與 war-meter 的兵力數字大部分時間定格不動,且停在整數(100,000、40,000、20,000),觀感不真實。
- 曹操烏林本陣在尾聲華容道敗走時,兵力數字應一路遞減(代表士兵逃亡、掉隊),目前只在 `t=240~420` 的 engage 插值窗內變動,其餘時間凍結。
- 參考站的兵力數字幾乎隨時都在滾動。

### 原因

`engine/timeline.js` 只有 `engage` 事件的 `losses` 會產生 troops 插值 key;插值窗短、目標值全是整數,窗外兵力恆定。

### 建議修法(兩層並進)

1. **引擎**(`engine/timeline.js` 編譯期):支援事件帶 `attrition` 欄位(建議語意:每章內分鐘的兵力流失量或流失率,擇一定義並註明),在 `rout` / `move`(敗走行軍)等事件編譯時轉成 troops 軌道的斜率 key。維持純函數模型。
2. **資料**(`battles/chibi/events.json`):
   - `losses` 目標值改為非整數觀感的數字(例:40,000 → 41,300 之類),插值窗(`t` ~ `t_end`)拉長,讓數字大部分時間在滾動。
   - 烏林本陣(`cao_camp`)華容道全程(`t=0~600` move)加上 attrition,讓敗走途中持續掉兵。
   - 第二章曹軍連環船隊潰走段同理。

### 驗收

- 播放第二章與尾聲,名牌與 war-meter 數字持續變動,無長時間定格整數。
- 任意拖曳/倒帶/跳章後數字與重播一致(確定性)。
- `.verify_phase5.py` 通過(結算損失數字若因資料調整變動,先更新斷言再跑)。
- console 無錯誤。

## 問題 4:地圖相對部隊太小,行軍一下子就走到、互相碰撞(先於問題 2 處理)

### 現象

- 部隊佔地與戰場尺寸比例失衡:艦隊陣列 6 欄 × 間距 24、列距 30(`engine/units.js`),單一艦隊約 **144×120 世界單位**,佔 1200×800 地圖的 12%~15%。
- 參考站(sekigahara.tenz.net)單一部隊約佔戰場寬度 2%~3%,地圖有「廣闊戰場」感;我們差 4~5 倍。
- 行軍距離以「部隊身位」計太短:黃蓋渡江全程 276 單位 ≈ 不到兩個艦隊身位,出發即抵達、名牌互相重疊,單位之間視覺上擠在一起。

### 建議修法(雙管齊下,目標比例改善 3~4 倍)

1. **縮小部隊佔地**(`engine/units.js`):陣列間距 24/30 → 約 14/18,幾何體(BOX_GEO 4×3×4、HULL_GEO 6×3×18)等比縮小約 0.6~0.7;旗幟、名牌錨點高度連動。
2. **放大地形**(`battles/chibi/battle.json` + `battles/chibi/events.json`):`terrain.size` 1200×800 → **2400×1600**,以下座標全部等比遷移:
   - battle.json:`hills[].pos/radius/height(height 不動)`、`landmarks[].pos`、`units[].spawn`
   - events.json:所有 `move.to`、`camera.pos`、`fire.pos`
3. **比例遷移後必須連動調整的引擎/資料參數**(漏調會破圖):
   - `events.json` 的 `atmosphere[].fog` 距離([1300,3600] 等是按舊尺寸調的)
   - `engine/director.js` 運鏡高度/距離(follow、overview、點名牌飛鏡頭的取景框)
   - `engine/scene.js` 相機初始位置、OrbitControls 距離限制、far plane
   - `engine/effects.js` 火攻/箭雨的範圍與粒子尺寸(burns 範圍按單位佔地、volley 弧線按距離)
   - `engine/terrain.js` 程序化貼圖解析度是否夠(放大後岸線細節)
4. **UI 畫面占比保持不變(使用者明確要求)**:名牌、地名標籤、字卡、war-meter、時刻面板、HUD 都是 DOM overlay,以螢幕像素定尺寸——**不得**因地圖放大而縮放任何 UI 的字級或尺寸,改完後 UI 佔畫面比例必須與現狀一致。唯一例外是世界空間的 3D 旗幟:單位幾何縮小後旗幟會跟著變小,需確認鏡頭常用距離下旗幟仍可辨識,必要時旗幟比例單獨調大。

### 驗收

- 同屏 overview 下,單一部隊佔畫面寬度比例明顯下降(目標感受向參考站靠攏)。
- **UI 佔畫面比例與改動前一致**(名牌/字卡/war-meter/時刻面板字級與尺寸不變,可用改動前後截圖對比)。
- 黃蓋渡江、華容道行軍等 move 有「跨越距離」的感覺,部隊之間不重疊、名牌不擠成一團。
- 三章完整播放 + 任意 seek:鏡頭取景正常(不穿地、不失焦)、霧不糊屏、火攻特效對齊船陣。
- `.verify_phase5.py` 通過,console 無錯誤。
