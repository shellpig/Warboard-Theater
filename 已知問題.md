## 問題 3:兵力數字常卡在整數,缺乏持續變動

### 現象

- 部隊名牌與 war-meter 的兵力數字大部分時間定格不動,且停在整數(100,000、40,000、20,000),觀感不真實。
- 曹操烏林本陣在尾聲華容道敗走時,兵力數字應一路遞減(代表士兵逃亡、掉隊),目前只在 `t=240~420` 的 engage 插值窗內變動,其餘時間凍結。
- 參考站的兵力數字幾乎隨時都在滾動。

### 原因

`engine/timeline.js` 只有 `engage` 事件的 `losses` 會產生 troops 插值 key;插值窗短、目標值全是整數,窗外兵力恆定。

### 建議修法(兩層並進)

1. **引擎**(`engine/timeline.js` 編譯期):支援事件帶 `attrition` 欄位(建議語意:每章內分鐘的兵力流失量或流失率,擇一定義並註明),在 `rout` / `move`(敗走行軍)等事件編譯時轉成 troops 軌道的斜率 key。維持純函數模型。
2. **資料**(`battles/chibi/events.json`):
   - `losses` 目標值改為非整數觀感的數字(例:40,000 → 41,300 之類),插值窗(`t` ~ `t_end`)拉長,讓數字大部分時間在滾動。
   - 烏林本陣(`cao_camp`)華容道全程(`t=0~600` move)加上 attrition,讓敗走途中持續掉兵。
   - 第二章曹軍連環船隊潰走段同理。

### 驗收

- 播放第二章與尾聲,名牌與 war-meter 數字持續變動,無長時間定格整數。
- 任意拖曳/倒帶/跳章後數字與重播一致(確定性)。
- `.verify_phase5.py` 通過(結算損失數字若因資料調整變動,先更新斷言再跑)。
- console 無錯誤。

## 問題 2:畫面上軍隊太少,只有本陣級單位

### 現象

- 全場僅 7 個單位,常駐可見只有 4~5 個(曹軍連環船隊、烏林本陣、周瑜水軍、黃蓋船隊、劉備軍本隊),戰場太空。
- 參考站同屏約 20 個部隊名牌。
- `battle.json` 的 `commanders` 與旁白文字中有戲份的將領(張遼、趙雲、張飛等)在地圖上沒有對應部隊。
- 尾聲「三道伏兵」(`events.json` 華容道章 `t=240` 一帶)目前只用旁白帶過,地圖上只有關羽一隊伏兵,趙雲、張飛接連截殺的橋段沒有演出來。

### 建議修法(只動資料,引擎支援任意單位數,不用改引擎)

拆分/新增單位(`battles/chibi/battle.json` 的 `units`),並把 `events.json` 各章的 move / engage / fire / rout 事件分配到新單位:

- **曹軍**:連環船隊拆出「文聘前軍」「毛玠・于禁水軍」;烏林本陣拆出「張遼隊」「徐晃隊」
- **孫吳**:周瑜水軍拆出「甘寧隊」「韓當・周泰隊」

注意事項:

- 約 8~10 個新單位;每隊需要 `id` / `faction` / `type` / 四語 `label` / `spawn` 座標 / `troops`。
- **兩側兵力總和維持 200,000 / 50,000**:拆出多少,本隊就扣多少。
- 既有 `losses` 目標值按拆分後的兵力重新計算(losses 是目標值,見共同前提)。
- 伏兵類單位開場用 `{ "type": "status", "unit": "...", "opacity": 0 }` 隱藏、現身時 `opacity: 1`(現有 `guanyu_force`、`strawboat_fleet` 有範例)。
