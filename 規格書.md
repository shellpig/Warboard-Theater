## 一、專案概念

網頁版「歷史戰役 3D 推演劇場」:

- 3D 戰場 + 時間軸驅動的戰況推演,體驗如「有導播的歷史紀錄片」
- 首頁為串流平台式「節目單」,海報卡片選擇戰役
- 一套引擎多場戰役:戰役資料(JSON + 素材)與引擎完全分離,新增戰役只需新增資料夾
- 後續用途:YouTube 頻道影片素材來源(運鏡考慮 16:9 錄製、UI 可隱藏)
- 第一場戰役:**赤壁之戰(演義版)**

## 二、技術定調

- 純前端、無 build step
- Three.js **r160**,jsdelivr CDN + importmap ES module,含 OrbitControls
- 程序化地形(高斯山丘高度場 + 頂點著色 + canvas 程序化貼圖),不使用 DEM
- 引擎 v1 即支援:陸地高度場 + **水面平面**(水戰為一等公民)
- 單位類型:`infantry`(陸軍方陣)/ `fleet`(船隊)
- 驗收方式:local server(如 `python -m http.server`)瀏覽器驗收
- 介面預設繁體中文,支援多語系(見第四節)

## 四、全域約定

### 座標系
- 世界單位平面座標 `[x, z]`,原點在戰場中心,**x = 東、z = 南**
- 心理比例 1 單位 ≈ 10 公尺(不嚴格)
- 赤壁戰場約 `1200 × 800`

### 時間系
- **章節制**:每章有獨立時間軸,事件 `t` = 章內分鐘偏移(number)
- 每章 `playback_sec` 定義 1× 速度下的實播秒數 → 各章自帶快轉倍率(鋪陳章大步快轉、決戰夜細刻度)
- 畫面顯示時間用 `time_display` 自由字串(支援「二更」等時辰),與排程解耦
- UI 時間軸為一條全域軸,以章節分段標記,可拖曳跨章、點章節名跳轉

### 多語系
- 支援:`zh-TW`(預設)/ `zh-CN` / `en` / `ja`
- 引擎 UI 字串:`engine/i18n.js` 內建四語字典
- 戰役內容字串:**內聯多語物件** `{"zh-TW":"…","zh-CN":"…","en":"…","ja":"…"}`
  - 引擎以 `pick(obj)` 解析,fallback 鏈:目標語系 → zh-TW → 物件第一個值
  - 純字串視為單語(等同只有 zh-TW)
- 語言切換器:節目單頁右上角 + 推演頁設定區;記 localStorage;`?lang=` 參數優先(錄影固定語言用)
- 演義 `quote` 原文:**任何語系都顯示漢文原句**(風味元素),en/ja 可選填譯文小字附於原句下

```json
          "t": 300, "t_end": 330,
          "time_display": { "zh-TW": "二更", "...": "..." },
          "type": "move",

          "to": [120, 80],
          "title": { "zh-TW": "黃蓋詐降船隊出發", "...": "..." },
          "camera_hint": "follow"
        }
      ]
    }
  ]

```

#### 事件 type 語意
| type | 必要欄位 | 語意 |
|---|---|---|
| `move` | unit, to, (t_end) | 單位自 t 起移動至 `to`,t_end 抵達(省略則瞬時) |
| `engage` | unit(s) | 進入交戰狀態;可帶 `losses`(兵力插值目標)|
| `fire` | unit / units | 指定對象起火(腳本驅動延燒:多個 fire 事件分批)|
| `defect` | unit | 詐降/倒戈,名牌閃爍 defect 徽章 |
| `rout` | unit | 潰走狀態,名牌淡化 |
| `status` | unit | 通用狀態/兵力修正(`troops`、`badge`)|
| `narration` | title/desc | 純旁白字卡(可帶 speaker/quote/fx)|
| `camera` | camera_hint/pos | 純鏡頭指令 |

- `camera_hint`:`follow / overview / none`(省略 = 導播自行判斷)
- `fx`(選填):事件附帶特效,`kind`:`volley / explosion / smoke / ring …`
- `speaker`(選填):commanders id → 字卡左側顯示頭像(3:4)+ 名字

## 六、執行期模型(重要)

**單位狀態 = 時間的純函數**(參考站驗證的設計):

1. 載入時,timeline.js 將 `move / rout / status / engage / fire` 事件**編譯為每單位的關鍵幀軌道**:位置 keys、兵力 keys、狀態 keys、透明度 keys
2. 執行期一律 `posAt(unit, t)` / `troopsAt(unit, t)` / `stateAt(unit, t)` 純函數取值
3. 因此時間軸可任意拖曳、倒帶、跳章,全場狀態永遠正確,且每次播放完全確定(對 YouTube 錄製重要)
4. `narration / camera` 類事件不進軌道,屬「表演層」,seek 時只取目前區間最後一張字卡
5. 章節在載入時銜接成一條全域播放軸(各章依 `playback_sec` 換算播放速率)

## 七、引擎功能清單

### 操作(比照參考站)
- 拖曳旋轉 / 滾輪縮放 / 右鍵平移(OrbitControls)
- Space 播放暫停、0.5×/1×/2×/4× 速度、時間軸拖曳、←→ 跳時間、章節跳轉
- 點擊部隊名牌 → 鏡頭飛往該隊;hover → tooltip(部隊介紹、兵力)

### 自動導播
- camera state machine:事件觸發鏡頭飛往現場(`camera_hint`)
- 使用者手動操作即暫停自動運鏡(manual hold),下一事件時導播接管

### UI 層(DOM overlay,worldToPx 投影)
- 部隊名牌:陣營色邊條 + 名字 + 兵力(隨 t 插值遞減)+ 狀態徽章
  - 徽章:進軍 / 交戰 / 潰走 / **詐降(閃爍)** / 撤離 / 本陣
- 地名標籤:依 `landmarks.type` 分樣式
- 旁白字卡(lower-third):title/desc + 選填 speaker 頭像 + 選填 quote 原文(小字)
- **war-meter**:雙方(按 side)總兵力即時數字 + 比例條
- 片頭 title screen、結算 end card(勝負、交戰時間、雙方損失、關鍵轉折、「↺ 重新觀看」)
- 電影感 overlay:膠片顆粒 + 暗角 + letterbox 黑邊(cinema mode)
- 語言切換器

### 大氣系統
- `skyColor(t)` / `fogDensity(t)` 隨模擬時間變化(赤壁:夜 → 黎明)
- 光照、霧、天色與章節時刻聯動

### 特效(effects.js)
- 火攻:**腳本驅動延燒**(fire 事件分批指定),GPU Points 粒子火焰 + 煙柱 + 水面火光反光,**粒子上限 1~2 萬**
- 箭雨曳光 `volley`(草船借箭)
- 煙、衝擊環等通用 fx

### 音效(後做)
- Web Audio 全合成,無音檔

### 錄影友善模式
- 一鍵隱藏 UI、鎖 16:9,與 letterbox 互補(優先度低)

## 八、美術方向

- **棋盤推演風**:部隊 = 簡潔幾何體陣列(步兵 = 方陣方塊群、船隊 = 低面數船形體排列,instanced)+ 陣營色旗幟(canvas 貼圖)
- 質感靠燈光、水面反射、火特效與電影感 overlay,不走寫實路線
- 地形:頂點著色 + canvas 程序化貼圖並用(長江岸線、水寨範圍畫進貼圖)
- 主帥肖像(使用者提供):**3:4 直式、胸像、去背 PNG、兩軍視線相對(一左一右)**,AI 生成統一畫風

## 九、第一場戰役:赤壁之戰(演義版)

### 已定事項
- 依《三國演義》敘事,介面標註「本推演依《三國演義》敘事」
- 地理採蒲圻說(今湖北赤壁市),長江江面為主戰場,北岸烏林為曹軍水寨
- 三陣營:曹操軍(north)vs 孫吳軍 + 劉備軍(south)

### 章節分鏡
1. **第一章 風起江上**(快轉,約數日):蔣幹盜書 → 草船借箭 → 苦肉計 → 借東風
2. **第二章 火燒赤壁**(決戰夜,細刻度 20:00 → 翌晨):黃蓋出發 → 詐降接近 → 火攻分批延燒 → 連環船陷火海 → 曹軍潰走
3. **尾聲 華容道**(鏡頭轉陸路):曹操敗走 → 三道伏兵 → 關羽義釋 → 結算畫面

### 內容備註
- quote 原文引句只放名場面(借東風、火燒連環船、華容道等),不是每事件都有
- 四語內容:初稿先寫 zh-TW,en/ja 譯文由使用者最終人工校對

## 十、開發階段
